# Спецификация — Smart Search Prototype

## Цель

Интерактивный веб-прототип механики умного поиска:
- Текстовый ввод разбирается в структурированные фильтры + lifestyle-хвост в реальном времени
- Двустороннее преобразование: текст → фильтры и фильтры → текст
- Подсветка распознанных фраз прямо в поле ввода

---

## Стек

- Vanilla JS, без фреймворков

## Архитектура

### parser.js — ядро логики

Два направления:

**`parseText(text) → { filters, matches, unmatched }`**
- Проходит по `SCHEMA` — словарный поиск triggers в тексте (longest-first, без пересечений)
- Применяет `RANGE_PATTERNS` — regex для числовых фильтров
- Собирает `unmatched` — слова, не попавшие ни в один фильтр, очищенные от стоп-слов

**`generateText(filters) → string`**
- Обратное: каждый фильтр → читаемая фраза через `genText` из схемы
- Результат подставляется в строку поиска при переключении из режима фильтров

### SCHEMA — фильтры

| filterId | Тип | Описание |
|---|---|---|
| `deal` | single | Купить / Снять |
| `rooms` | multi | Студия, 1–6+ комнат |
| `district` | single | 16 районов Москвы |
| `type` | single | Вторичка / Новостройка |
| `balcony` | multi | Балкон / Лоджия |
| `renovation` | multi | Без ремонта / Косметический / Евроремонт / Дизайнерский |
| `house_type` | multi | Кирпичный / Монолитный / Панельный / Сталинский |
| `floor_options` | multi | Не первый / Не последний / Последний |
| `bathroom` | multi | Совмещённый / Раздельный / Два и более |
| `parking` | multi | Подземная / Наземная / Многоуровневая |

### RANGE_PATTERNS — числовые фильтры

| filterId | Примеры распознавания |
|---|---|
| `price_max` | `до 12 млн` |
| `price_min` | `от 5 млн` |
| `price_max_k` | `до 80 тыс` (аренда) |
| `price_min_k` | `от 50 тыс` (аренда) |
| `area_min` | `от 50 м²`, `50 м²` |
| `area_max` | `до 80 м²` |
| `floor_min` | `от 3 этажа` |
| `floor_max` | `до 10 этажа` |
| `metro_time` | `10 минут пешком` |

Порядок паттернов важен: `area_max` идёт раньше `area_min`, `price_max_k`/`price_min_k` — раньше млн-паттернов (предотвращает ложные срабатывания).

## UI — results.html

### Два режима (переключатель «текст / фильтры»)

**Текст-режим:**
- Поле ввода с highlight backdrop — распознанные слова подсвечиваются `<mark>`, нераспознанные — классом `.unmatched`
- Предлоги автоматически включаются в подсветку совпадения («в Марьино» → вся фраза)
- Suggestions — динамические подсказки на основе заполненных фильтров (до 12 штук)

**Режим фильтров:**
- Chip-группы для каждого filterId из SCHEMA
- Числовые поля: `priceMinInput` / `priceMaxInput` (млн), `priceMinKInput` / `priceMaxKInput` (тыс), площадь, этаж, метро
- Поле «Без фильтра» — lifestyle-хвост (`unmatched`)
- Кнопка «Найти» → `results.html?q=...`

### Видимость блоков цены

Блоки цены зависят от выбранного типа сделки:

| `deal` | Цена покупки (млн) | Цена аренды (тыс/мес) |
|---|---|---|
| `buy` | показывается | скрыта |
| `rent` | скрыта | показывается |
| не выбран | оба блока видны | оба блока видны |

Реализовано в `renderFilterPanel()` через `#priceBuyGroup` / `#priceRentGroup`.

### Suggestions

Порядок предложений определяется состоянием фильтров:

| Состояние | Порядок suggestions |
|---|---|
| Строка заканчивается на «в» | district, type, renovation… |
| Есть фильтр цены, нет района | district, затем остальные |
| deal не выбран | deal |
| rooms не выбраны | rooms, district, type… |
| district не выбран | district, type… |
| остальные случаи | type, renovation, house_type… |

## Ограничения прототипа

- Только мобильная версия (viewport 375px)
- `results.html` — заглушка, редирект по `q=` query param
- Без бэкенда и реальной базы объявлений
